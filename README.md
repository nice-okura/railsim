# 🚂 レールランド

子供向けのレール配線＆電車走行シミュレーター。ブラウザだけで動く静的 Web アプリです。

## あそびかた

- 左の「パーツばこ」からレールや電車をドラッグして配置
- レールの端は **凸と凹のコネクター** になっていて、近づけるとカチッと吸着（凸は凹としか繋がりません）
- **ぶんきレール** はクリックで進路を切り替え（緑の矢印が進む方向）
- 「▶️ うごかす」で走行開始。電車は複数台置けて、ぶつかると 💥 衝突します（🚑 なおす で復旧）
- レールをタップで選択 → 🔄 まわす / 🗑 けす。電車は ↩️ ぎゃくむき も可能
- 何もない場所のドラッグで画面移動、ホイール／ピンチでズーム
- レイアウトは自動でブラウザに保存されます

## レールの種類を追加するには

`app.js` の `registerRail()` を呼ぶだけで、パレットにも自動で追加されます。

```js
registerRail("long", makeStraightType("ながい", 220));
```

任意形状は `endpoints`（凸凹と向き）と `paths`（s→座標の関数）を持つ定義オブジェクトを登録します。

## ローカルで動かす

```sh
python3 -m http.server 8000
# → http://localhost:8000
```

## テスト（開発者向け）

ヘッドレスブラウザによる回帰テストがあります（静的サーバー内蔵）。

```sh
cd tests
npm run setup   # 初回のみ
npm test        # 🎉 ALL PASS になればOK
```

起動・スナップ接続・凹凸の極性・分岐・折り返し・衝突・保存/復元を検証し、
`tests/screenshots/` にスクリーンショットを出力します。

アーキテクチャの詳細と開発ルールは [CLAUDE.md](CLAUDE.md) を参照してください。
（AI アシスタントで開発する場合は自動で読み込まれます。レール追加・検証の手順は
`.claude/skills/` にスキルとして用意してあります。）

## GitHub Pages で公開する

1. このリポジトリを GitHub に push
2. リポジトリの Settings → Pages → Source を「Deploy from a branch」、Branch を `main` / `(root)` に設定
3. `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

ビルド不要（HTML/CSS/JS のみ）なので、そのまま公開できます。
