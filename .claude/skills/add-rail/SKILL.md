# レールの種類を追加する

レールは `app.js` の `RAIL_TYPES` レジストリで管理されている。
`registerRail(id, def)` を1回呼ぶだけで、**パレットへの表示・サムネイル描画・保存/復元・スナップ接続**まで全部自動で効く。

## 手順

1. `app.js` の「レール種類レジストリ」セクションに定義を追加する。

   **既存ファクトリで済む場合**（長さ違いの直線など）は1行:

   ```js
   registerRail("long", makeStraightType("ながい", 220));
   ```

   **新しい形状の場合**は定義オブジェクトを書く:

   ```js
   registerRail("myRail", {
     label: "〇〇（ひらがなで・パレットに表示される）",
     endpoints: [
       // out は「外向き」の角度。pol: 'F'=凹, 'M'=凸
       { x: 0, y: 0, out: Math.PI, pol: "F" },
       { x: 100, y: 0, out: 0, pol: "M" },
     ],
     paths: [
       // pos: s(0〜len) → ローカル座標 {x, y, a}。a は進行方向の角度
       // a/b はこのパスの始端/終端の endpoints 番号
       { len: 100, pos: s => ({ x: s, y: 0, a: 0 }), a: 0, b: 1 },
     ],
   });
   ```

2. `PALETTE_ORDER` は `registerRail` が自動で積むので触らなくてよい。
   並び順を変えたいときだけ登録順を入れ替える。

## ルール（守らないと既存機構が壊れる）

- **座標系は y 軸下向き**。カーブの `sign=+1` は画面下方向（右曲がり）。
- 端点の `out` は必ず**レールの外を向く**角度にする（接続判定は「向きが正反対」で見るため）。
- 凹凸の慣例: 進入側(始端)が `F`(凹)、出口側(終端)が `M`(凸)。
- `pos(0)` と `pos(len)` の座標・角度は、対応する endpoint の位置・向きと**厳密に一致**させる
  （`pos(len).a` = 終端 endpoint の `out`、始端は `out` が `pos(0).a + π`）。
  ズレていると電車が乗り継ぎ時にワープする。
- 分岐系（パスが複数で共通の基端を持つもの）は `isSwitch: true` を付ける。
  paths[0] が `sw=0`（デフォルト直進）、paths[1] が `sw=1`（分岐側）。
- 曲線は `arcPath(半径, 角度, sign)`、直線は `straightPath(長さ)` を再利用する。
  幾何定数（`STRAIGHT_LEN=110`, `CURVE_R=140`, 45°）に整合する寸法にすると既存レールと組み合わせやすい。
- ラベルはひらがな中心。商標・既製品を連想させる名前を付けない。

## 追加後の確認

1. `tests/smoke.js` の「[1] 起動」テストはパレット枚数を `>= 7` で見ているのでそのまま通るが、
   新レール固有の挙動（分岐・特殊な接続など）があればテストを追加する。
2. `/verify` スキルの手順で回帰テストとスクリーンショット目視を行う。
   特に `tests/screenshots/final.png` とパレットのサムネイルの見た目を確認すること。
