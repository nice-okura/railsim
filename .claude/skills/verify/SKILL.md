# レールランドの動作検証

## 手順

1. **構文チェック**

   ```sh
   node --check app.js
   ```

2. **回帰テスト**（静的サーバー内蔵。別途サーバー起動は不要）

   ```sh
   cd tests
   npm run setup   # 初回のみ: npm install + ヘッドレスブラウザ導入
   npm test
   ```

   最後に `🎉 ALL PASS` と出れば成功。終了コード 0。
   失敗した場合は ❌ の行に期待値と実際値が出る。

3. **スクリーンショットの目視確認**（見た目に関わる変更をした場合は必須）

   テストが `tests/screenshots/final.png`（全体）と `crash.png`（衝突時）を出力するので、
   Read ツールで画像を開いて確認する:
   - レール（道床・まくらぎ・レール溝）が崩れていないか
   - パレットのサムネイルが全種類描画されているか
   - 電車の向きが進行方向と合っているか
   - 木製レール風の配色が保たれているか（既製品玩具を想起させる配色になっていないか）

4. **新しい挙動を追加した場合**は `tests/smoke.js` にテストを追加してから再実行する。

## テストを書くときの注意

- アプリの状態（`pieces` / `trains` / `view` / `connMap` など）は script のトップレベル `let` なので
  `page.evaluate()` から直接読み書きできる。
- 状態を書き換えたら `connDirty = true` を忘れない。
- **クリック座標は必ず `worldToScreen()`（smoke.js 内のヘルパー）で変換する。**
  canvas はパレット分オフセットされ、パン/ズームもあるため、生の座標では別の場所を叩いてしまう。
- 決定的なテストを優先する（`stepTrain` を直接呼ぶ等）。実時間走行の待ちは衝突テストなど最小限に。

## 手動確認したいとき

```sh
python3 -m http.server 8000
# → http://localhost:8000 をブラウザで開く
```
